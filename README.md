# BAML Diligence Eval

A controlled benchmark and adversarial stress test of [BoundaryML BAML](https://docs.boundaryml.com/) against a raw OpenAI + Pydantic baseline for structured LLM output.

## Summary

This repository contains a from-scratch evaluation of BAML's structured output capabilities, built as part of a VC technical diligence exercise. The benchmark ran 1,800 LLM calls across two models (Claude 3 Haiku and GPT-4o-mini) and two prompt strategies, comparing BAML's Schema-Aligned Parsing against manual JSON Schema injection with Pydantic validation. BAML achieved a 100% parse success rate (900/900). The Pydantic baseline achieved 87.1% (784/900), with all 116 failures falling into a single failure mode: schema-echo, where the LLM mirrors the `$defs`/`properties` JSON Schema wrapper from the prompt instead of producing conforming data. The root cause is the prompt format, not the parser. BAML injects a 13-line human-readable pseudo-schema via `{{ ctx.output_format }}`, while Pydantic's `model_json_schema()` injects a 62-line raw JSON Schema that smaller models confuse for the expected output structure.

## Setup

```bash
git clone https://github.com/thisisvk45/baml-diligence-eval.git
cd baml-diligence-eval

python3 -m venv .venv
source .venv/bin/activate

pip install baml-py==0.222.0 openai pydantic python-dotenv

cp .env.example .env
# Edit .env and add your OpenRouter API key

# Generate the BAML typed client
baml-cli generate
```

Requires Python 3.11 or later and an [OpenRouter](https://openrouter.ai/) API key with access to `anthropic/claude-3-haiku` and `openai/gpt-4o-mini`.

## How to Reproduce

### 1. Run the 1,800-call benchmark

```bash
python test_harness_100.py
```

This runs 100 trials per cell across 3 test cases, 2 approaches (BAML vs Pydantic), and 3 configurations (original-Haiku, hardened-Haiku, original-gpt4o-mini). Results are written to `results.csv` with checkpoint recovery. Expected runtime is approximately 95 minutes. Estimated cost is approximately $0.71.

### 2. Run the adversarial stress test

```bash
python adversarial_agent.py
```

Generates 20 adversarial test cases via GPT-4o-mini using BAML, then runs each case through both BAML and Pydantic pipelines targeting Claude 3 Haiku. Results are saved to `adversarial_results.json`. Expected runtime is approximately 2 minutes. Estimated cost is approximately $0.07.

### 3. View the raw data

- `results.csv` contains all 1,800 rows from the benchmark.
- `pydantic_failures.txt` contains every Pydantic failure with the raw LLM response that caused it.
- `adversarial_results.json` contains per-case outcomes from the adversarial run.
- `FINAL_REPORT.md` contains the full analysis with tables and latency percentiles.

## Findings

Across 900 BAML calls, the parse success rate was 100%. Across 900 Pydantic calls, the parse success rate was 87.1%, with 116 failures. Every single failure was the same mode: schema-echo. The LLM returned a response wrapping the actual data inside the `$defs` and `properties` keys from the Pydantic JSON Schema, producing structurally valid JSON that did not conform to the expected top-level schema. BAML's Schema-Aligned Parsing recovered from these malformed responses automatically.

The root cause is the prompt format, not the validation library. Pydantic's `model_json_schema()` produces a 62-line JSON Schema block with nested `$defs`, `properties`, `required`, and `title` keys. When Claude 3 Haiku encounters this in the prompt, it sometimes mirrors the schema structure back as part of its response. BAML avoids this by injecting a compact, human-readable pseudo-schema that smaller models parse correctly. GPT-4o-mini did not exhibit schema-echo behavior with either prompt format.

A hardened Pydantic prompt (explicitly instructing "no markdown, no prose, just raw JSON") eliminated all failures, achieving 100% across all configurations. This confirms the failure is prompt-induced, not a fundamental parsing limitation.

The adversarial stress test generated 18 attack cases targeting seven failure modes (schema-echo, truncation, type coercion, field omission, prose preamble, markdown fencing, hallucinated fields). BAML passed 18/18. Pydantic passed 17/18, with the single failure being schema-echo on an empty-input case. No new failure modes were discovered.

## Methodology

- **Models**: `anthropic/claude-3-haiku` and `openai/gpt-4o-mini`, both accessed via OpenRouter
- **Sample size**: 100 trials per cell (3 test cases x 2 approaches x 3 configs = 18 cells, 1,800 total calls)
- **Confidence intervals**: 95% Wilson score intervals on pass rates
- **Temperature**: 0.2 for all calls
- **No retries**: Failures are recorded as-is to measure raw reliability
- **Adversarial generation**: 18 cases generated by GPT-4o-mini via BAML, tested against Claude 3 Haiku

## Limitations

- Single task domain (job description to resume bullet matching). Results may not generalize to other structured output tasks.
- Two models tested. Behavior on other model families (Llama, Mistral, Gemini) is unknown.
- Single provider (OpenRouter). Provider-level caching or routing could affect results.
- Sample size (n=100 per cell) is adequate for detecting high-frequency failure modes but insufficient for rare failures below approximately 3%.
- Latency comparisons are confounded by token count differences between BAML and Pydantic prompts (the Pydantic prompt includes a 62-line JSON Schema block).
- The adversarial generator used the same model family (GPT-4o-mini) for case generation, which may limit the diversity of attack patterns.

## Context

This evaluation was built as part of a VC technical diligence exercise for Basis Set Ventures, assessing BoundaryML's BAML framework for a potential investment. The full investment memo with thesis, risk analysis, and recommendations lives separately from this repository. This repo contains only the technical benchmark, raw data, and reproducible code.
